# 🗿 BECOME GIGACHAD — LIVE EDITION

Соревновательный трекер развития **для двоих** (работает и для 3–4 человек) в стиле Minecraft-достижений. Вы с другом видите дни друг друга, кидаете друг другу задания и ачивки, зарабатываете XP и соревнуетесь — кто быстрее пройдёт путь от НПС до ГИГАЧАДА.

## Что умеет

- **«Система видит, что ты появился»** — первый заход в день фиксируется автоматически («🌅 в сети с 08:12»), друг получает уведомление.
- **Привычки-чеклисты** — зубы, зарядка, отжимания, чтение… Настраиваются под себя, каждая даёт очки. Отметки — только за сегодня (задним числом накрутить нельзя).
- **🏆 Ачивки (Minecraft advancements)** — 5 редкостей (🪨10 → 👑250 XP), прогресс по шагам, ежедневные 🔁 (сбрасываются каждый день), общие 🤝 (XP каждому) и гонки 🏁 (XP первому), дерево-блокировки 🔒 (потомок открывается после родителя), «вернуть» ↩. Иконка из 35 пиксельных или эмодзи, 🎲 кубик подкидывает идею из 100 готовых. XP вливается в общие очки.
- **Лента дня** — еда (фото + калории), взвешивания (фото + кг), работа, заметки. Весь день как хроника, видимая другу.
- **ИИ-подсчёт калорий по фото** (опционально) — сфоткал тарелку, Claude оценил калории и назвал блюдо. Так же читает вес с фото дисплея весов.
- **Задания** — себе (+15) и вызовы другу (+25). Просрочка подсвечивается.
- **Вечерний отчёт** — текст + автосводка дня (очки, привычки, ккал, задачи). +20 очков.
- **Чат** — в реальном времени, с фото.
- **Геймификация** — очки за всё, «Дуэль недели» 👑, стрики 🔥, уровни НПС → Дрищ → Качок → … → ГИГАЧАД, графики очков/веса/калорий, 8-битные звуки.
- **Minecraft-интерфейс** — пиксельный шрифт Press Start 2P, каменные таблички, хотбар-навигация, тосты «Достижение получено!». Адаптив под ПК и телефон.
- **PWA** — ставится на телефон как приложение (Android и iOS), тёмная тема.

> Изначально назывался «Тандем». Механики ачивок перенесены из проекта Achievements (иконки, звуки, 100 идей — `public/icons.js`, `sound.js`, `ideas.json`).

## Быстрый старт

Нужен **Node.js 24 LTS** (минимум 22.13 — используется встроенный SQLite).

```bash
npm install
npm start
# → http://localhost:3000
```

Первый зашедший регистрируется, потом регистрируется друг. По умолчанию регистрация закрывается после 4 аккаунтов (`MAX_USERS`).

Все данные — в папке `data/` (база SQLite + фото). Удалил папку — начал с чистого листа.

## Настройка

Скопируй `.env.example` в `.env` и поправь под себя:

| Переменная | Что делает |
|---|---|
| `PORT` | Порт сервера (по умолчанию 3000) |
| `TZ` | Часовой пояс, например `Europe/Moscow`. **Важно**: определяет, когда наступает «новый день» |
| `MAX_USERS` | Лимит аккаунтов (по умолчанию 4) |
| `INVITE_CODE` | Если задан — без кода не зарегистрироваться |
| `ANTHROPIC_API_KEY` | Ключ для ИИ-подсчёта калорий/веса по фото. Без ключа всё работает, ввод вручную |
| `AI_MODEL` | Модель анализа фото. По умолчанию `claude-opus-4-8` (точнее), дешевле — `claude-haiku-4-5` |
| `DATA_DIR` | Куда складывать базу и фото |

Ключ API берётся на [platform.claude.com](https://platform.claude.com). Один анализ фото стоит копейки, но следи за лимитами.

## Деплой на VPS

1. Поставь Node 24 (через [nodesource](https://github.com/nodesource/distributions) или `nvm`).
2. Залей папку проекта, `npm install`, создай `.env` (как минимум `TZ`).
3. Systemd-юнит `/etc/systemd/system/tandem.service`:

```ini
[Unit]
Description=Tandem
After=network.target

[Service]
WorkingDirectory=/opt/tandem
ExecStart=/usr/bin/node --disable-warning=ExperimentalWarning server.js
Restart=always
User=www-data
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable --now tandem
```

4. HTTPS обязателен для PWA и камеры на телефоне. Проще всего — [Caddy](https://caddyserver.com) (сам получает сертификат):

```
# /etc/caddy/Caddyfile
tandem.example.com {
    reverse_proxy localhost:3000
}
```

Или nginx + certbot — стандартный reverse proxy на `localhost:3000` (websocket-заголовки `Upgrade`/`Connection` не забудь для Socket.IO).

**Нет домена?** Используй [sslip.io](https://sslip.io): адрес вида `159-69-153-135.sslip.io` автоматически резолвится в IP `159.69.153.135` (подставь свой, точки → дефисы). Впиши такой хост в Caddyfile — Caddy получит на него настоящий Let's Encrypt сертификат, и HTTPS/PWA работают без покупки домена. В `.env` поставь `HOST=127.0.0.1`, чтобы наружу торчал только Caddy.

5. **Бэкап** — это просто папка `data/`: `tar -czf backup.tgz data/` по крону.

## Установка на телефон (PWA)

- **Android (Chrome)**: открой сайт → меню → «Добавить на главный экран» / «Установить приложение».
- **iOS (Safari)**: открой сайт → «Поделиться» → «На экран “Домой”».

Получишь полноэкранное приложение с иконкой. Фото еды снимается прямо камерой из приложения.

## Структура

```
server.js        # весь бэкенд: Express + Socket.IO + node:sqlite + Claude API
public/
  index.html     # каркас
  app.js         # вся логика клиента
  style.css      # тёмная тема
  manifest.json  # PWA
  sw.js          # service worker (офлайн-оболочка)
data/            # создаётся сама: tandem.db + uploads/ (в .gitignore)
```

## Очки (можно править в server.js)

| Действие | Очки |
|---|---|
| Привычка | 5–25 (настраивается) |
| Своя задача выполнена | +15 |
| Задача от друга выполнена | +25 |
| Приём пищи / взвешивание | +5 |
| Запись о работе | +10 |
| Заметка | +2 |
| Отчёт за день | +20 |

Стрик 🔥 засчитывается, если за день набрано ≥30 очков **или** написан отчёт.

## Роадмап (что дальше)

- **Умные часы** — напрямую браузер к часам не подключится; путь такой: нативная оболочка (Capacitor) → Health Connect (Android) / HealthKit (iOS) → POST шагов/сна/пульса в `/api/posts`. API уже готов это принять.
- **Мобильные приложения** — текущий PWA уже ставится на телефон; если захочется в сторы — обернуть в [Capacitor](https://capacitorjs.com) практически без изменений кода.
- **Push-уведомления** — Web Push поверх существующего service worker (нужен HTTPS, он и так будет).
- Напоминание «напиши отчёт» вечером, еженедельные итоги дуэли, ставки на челленджи.

## Безопасность

Прототип для своих: пароли хэшируются (bcrypt), сессии в httpOnly-куках, фото отдаются только залогиненным, регистрация ограничена лимитом/инвайт-кодом. Не выставляй без HTTPS.
